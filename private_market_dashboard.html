The Problem: private assets have no price between events

A private company is valued only when something happens — a funding round, a periodic mark-to-market by its holders, or a secondary transaction. Between those points there is no observable price and no liquidity. The valuation series is a step function: flat for months, then a jump. You cannot trade a step function.

Why the step function is un-tradable

  • No intraday price. There is nothing to quote against between events; a market maker cannot hedge or mark a position.
  • Stale marks. The “current” valuation can be a year old. Reported private-market NAVs are smoothed and lag public markets, understating true volatility and correlation.
  • No liquidity between events. Exit requires a negotiated secondary at an unknown discount, on weeks-to-months timelines, often with transfer restrictions and issuer consent.
  • Jump risk. All price discovery is compressed into the event itself — a down round reprices the whole holding period at once.
The task is therefore a transformation problem: convert a sparse, stepwise valuation series into a continuous return stream that can actually be bought, sold and risk-managed daily. Tab 02 shows how.

Constructing a Liquid Series

The transformation has three stages: take the discrete marks, model the continuous return path they imply, then map that return profile onto a basket of liquid listed equities so the series becomes genuinely tradable. Use the stage buttons to build the picture up.

The three steps in plain English

A

Take the discrete valuation points

Funding rounds, holder marks and secondary prints are the only real observations. Everything else is inference — so keep them distinct and honest: these are the anchors, nothing between them is data.

B

Model a continuous return path between anchors

Between two marks, estimate the path the valuation most plausibly followed — a bridge in log-space, conditioned on both endpoints, with volatility borrowed from comparable listed names (de-smoothed, since reported private marks understate risk). This yields a daily return series, but it is a model, not a price.

C

Map the return profile onto liquid listed equities

Find the basket of public stocks — matched on sector, stage and style (size, growth, quality, beta) — whose combined daily return best replicates the modelled path. The basket is tradable: real prices, real depth, daily dealing. The private return profile now has a liquid expression.

Where the Liquidity Comes From

Wrap the liquid proxy basket in an index or ETF and the liquidity is layered. The screen volume you see quoted is the smallest layer. The real capacity sits underneath, in the aggregate depth of the listed constituents, accessed through creation and redemption.

An AP can create or redeem against the basket at any size the constituents can absorb. Judging an index wrapper by its own printed volume is like judging a currency pair by the turnover of one broker.

The direct answer to the question I was asked

True liquidity = the underlying listed basket’s depth, not the ETF’s screen volume.
  • Primary layer: APs create and redeem shares in-kind against the basket, so the vehicle’s capacity scales with the constituents’ tradability, not with its own trading history.
  • Secondary layer: exchange order book plus RFQ/OTC — market makers will quote block size because they can hedge instantly in the underlying names.
  • No liquidity mismatch: because the holdings are liquid public equities, daily dealing is honest — there is no gate, no side-pocket, no fire-sale risk of the kind that breaks open-ended private-asset funds.

Link to GSPE — the GS/MSCI World Private Equity Return Tracker

Everything in tabs 01–03 is exactly what GSPE does at index scale: it holds liquid public equities selected to replicate the return profile of MSCI’s private-equity dataset (~$7.7tn of private-capital fund data) by region, sector, size and style — PE-like exposure with daily liquidity and no liquidity mismatch.

The GSPE pipeline

Steps two and three are the index-scale versions of Stage B and Stage C in tab 02: characterise the return profile implied by sparse private marks, then hold the liquid basket that best expresses it.

What it captures

  • The systematic return profile of private equity — the region, sector, size and style exposures that drive most of the asset class’s return, sourced from MSCI’s dataset rather than a hand-picked comparables list.
  • Daily, honest liquidity. The holdings are public equities, so dealing terms match the assets — the structural failure mode of semi-liquid private wrappers (gates, queues, stale NAVs) is designed out.
  • Immediate, cheap deployment. No capital calls, no J-curve, no vintage lottery; useful both as core exposure and as a liquidity sleeve alongside true fund commitments.

What it honestly does not capture

The illiquidity premium. Whatever extra return investors earn specifically for locking capital up cannot be replicated by liquid instruments — replication targets the return profile, not the lock-up compensation.

Manager selection. Dispersion between top- and bottom-quartile PE managers is enormous; GSPE deliberately delivers the asset-class return, not access to any specific GP’s alpha, deal flow or operational value-add.

Full mark-to-market volatility. The tracker will look more volatile day-to-day than smoothed private NAVs — that is a feature (honest pricing), but it changes the reported-risk optics some allocators are used to.

My Proposal: optimise the basket for executable liquidity, not just return fit

An original extension, clearly labelled as such. Replication optimisers typically minimise tracking error to the target return profile. I would add a liquidity-scoring layer that ranks candidate proxy names by basket tradability and tilts the replication toward the most tradable expression of the same return profile.

Mechanism Original proposal — illustrative numbers

1

Score every candidate name

Composite liquidity score per stock from: quoted bid–ask spread (bps), average daily volume in dollars, and listing depth (order-book size at touch, venue count, index-membership flow). Standardise each input cross-sectionally, combine with fixed weights, re-score monthly.

2

Tilt the optimiser

Instead of minimise TE, solve minimise TE − λ × basket liquidity score, subject to a hard tracking-error budget. λ sets how hard the basket leans toward tradable names; the TE budget guarantees the return profile is preserved.

3

Substitute like-for-like

Where two names carry near-identical factor exposure (same sector, size, style), the score breaks the tie toward the cheaper-to-trade one. Liquidity is bought where it is nearly free in fit terms — not by abandoning the profile.

The trade-off, made explicit Modelled, illustrative

Moving along the frontier from λ=0: a marginally looser return fit (tracking error rises tens of basis points) buys materially better executability (basket liquidity score rises steeply at first). The knee of the curve is the sensible operating point; beyond it, liquidity gains flatten while fit degrades.

How I would validate it

  • Historical tracking-error analysis. Backtest the liquidity-tilted basket against the untitled baseline over multiple regimes (2020 drawdown, 2022 rate shock, quiet markets). Confirm realised TE stays inside the budget and the fit loss is genuinely marginal.
  • Turnover and cost analysis. Measure rebalance turnover and estimated implementation cost (spread + impact) for both baskets. The tilted basket should show materially lower all-in trading cost — that saving is real return, and partially offsets the TE give-up.
  • Stress executability. Model liquidation of a large position in both baskets under stressed depth (e.g. halved ADV, doubled spreads). The tilted basket should exit faster and at lower cost — the scenario in which this layer earns its keep.
  • Capacity mapping. Estimate maximum vehicle size at acceptable daily participation rates. The liquidity tilt should raise strategy capacity — directly relevant to an index designed to scale.
Why this fits the GSPE thesis: the product’s core promise is liquidity without mismatch. Optimising the basket for executable liquidity — not just return fit — hardens exactly the property the product is sold on.

What the backtest actually shows

  • Daily factor regression fails completely (R² ≈ 0.00). OPAI.PVT is flat for weeks then jumps — marks do not respond to daily market moves, so daily betas are unidentifiable. This is tab 01’s stale-marks problem, measured on real data.
  • At monthly horizon a signal appears, but α dominates. Fitted idiosyncratic drift is ~40–50%/yr; factor betas are small and unstable across horizons and lags. Most of OpenAI’s valuation path over this window is company-specific, not systematic.
  • Implication for replication: a factor basket can carry the systematic part of a private name and the smoothing/cadence structure — it cannot manufacture single-name α. That is precisely why GSPE replicates a broad private-equity asset class (where idiosyncratic stories diversify away and region/sector/size/style dominate) rather than one company.
Try it: press Fit to data and watch α absorb the return while betas stay small. Then set α to 0 — the factor-only model cannot keep up with the marks. One private company is mostly idiosyncratic; a portfolio of thousands is mostly factors.
